Kick start

The Angrybee Web design is responsive and dark oriented.

We propose 2 CSS adapted to screen sizes (PCs, Tablets, Smartphones):

Screen size CSS file
< 576px (Smartphones) angrybee-dark-mobile.css
≥ 576px (Tablets, PCs) angrybee-dark-pc.css

Angrybee CSS declaration in HTML

						
<link rel="stylesheet" media="screen and (max-width: 575px)" href="$url/angrybee-dark-mobile.css">
<link rel="stylesheet" media="screen and (min-width: 576px)" href="$url/angrybee-dark-pc.css">

We propose also a set of Javascript libraries, designed by Angrybee but also by other Open Source projects to enhance the power of our framework and avoid reinventing the wheel.

Javascript files declaration

							
<header>
	...
	<script src="$url/highlight.min.js"></script>
	<script src="$url/anchor.min.js"></script>
	<!--Boostrap bundled with Popper-->
	<script src="$url/bootstrap.bundle.min.js"></script>
	<script src="$url/preloadjs.min.js"></script>
	<script src="$url/aos.js"></script>
	<script src="$url/kute.min.js"></script>
	<script src="$url/vivus.min.js"></script>
	<script src="$url/angrybee.js"></script>

	<script>
	//Enable syntax highlight
	hljs.highlightAll();



	</script>		
</header>
<body>
...
	<!-- End of file -->
	<script>
		AOS.init();	
	</script>
</body>

You just have to adapt the $url path to adapt the code sample you can copy/paste.


CSS Details

Angrybee has built its own CSS classes to propose a consistent library for its Dark Mode.

However, Angrybee has integrated other interesting CSS libraries to enhance its Framework capabilities and avoid reinventing the wheel.

Angrybee integrates the following CSS files:


Angrybee

Apache 2 License

Angrybee CSS Framework.

angrybee-dark-pc.css
angrybee-dark-mobile.css

Angrybee Symbol

Apache 2 License

Angrybee CSS dedicated to symbols.

angrybee-symbols.css

Github syntax highlight

BSD-3-Clause License

CSS for syntax highlights. See highlightjs. CSS associated to highlight.min.js

github-dark.min.css

Magic

MIT License

A CSS for animations with special effects. We choose this CSS because of its specificity regarding the special effects (see Magic website and its documentation).

magic.min.css

Bootstrap

MIT License

Bootstrap framework bundled with Popper. See Bootstrap.

bootstrap.min.css

Bootstrap Icons

MIT License

Predefined Bootstrap icons. See Bootstrap icons.

bootstrap-icons.css

Hover.css

No License

CSS for animations dedicated to the "hover" event. See Hover.css.

bootstrap-icons.css


You don't need to include these CSS files in your code, they are already included in the Angrybee CSS files.

CSS included by default

							
@import url("$url/github-dark.min.css");
@import url("$url/bootstrap-icons.css");
@import url("$url/bootstrap.min.css");
@import url("$url/angrybee-symbols.css");
@import url("$url/magic.min.css");
@import url("$url/aos.css");
@import url("$url/hover-min.css");

Javascript Details

Some extra-CSS are delivered with Javascript so it is mandatory to get them. The Angrybee Framework is bundled with all the necessary to run these extra-libraries..

But you need to call them in your HTML page.

By default, we don't modify the source code of the CSS and JS projects we include. In some cases we add classes or we override styles.

The Javascript libraries included are:

AnchorJS

MIT License

preloadjs.min.js

AOS

MIT License

Animate On Scroll library is a simple but powerfull animation library to manage appearence of elements during scrolling. See AOS.

KUTE.js

Vivus

MIT License

Dark Mode

Angrybee CSS is made for specific Dark Mode.

Simply add ab-dark-- to the class attribute of your body page to enable this mode.

Enable Dark mode

					
<body class="ab-dark--">
<!-- content of the page -->
<body>

See Color section for additional information.

Padding

Padding is space around an element's content, inside of any defined borders.

We have 3 category of classes.

  • classes for top and bottom padding
  • classes for top padding
  • classes for bottom padding

For a top and bottom padding:

  • ab-padding-16: add 16px top + bottom padding
  • ab-padding-24: add 24px top + bottom padding
  • ab-padding-32: add 32px top + bottom padding
  • ab-padding-48: add 48px top + bottom padding
  • ab-padding-64: add 64px top + bottom padding

For a top padding only:

  • ab-padding-top-16: add 16px top padding
  • ab-padding-top-24: add 24px top padding
  • ab-padding-top-32: add 32px top padding
  • ab-padding-top-48: add 48px top padding
  • ab-padding-top-64: add 64px top padding

For a bottom padding only:

  • ab-padding-bottom-16: add 16px bottom padding
  • ab-padding-bottom-24: add 24px bottom padding
  • ab-padding-bottom-32: add 32px bottom padding
  • ab-padding-bottom-48: add 48px bottom padding
  • ab-padding-bottom-64: add 64px bottom padding

Margin

Margin is space around an element's content, outside of any defined borders.

We created Margin classes of 16px, 24px, 32px, 48px and 64px.

  • Class ab-margin-16: creates a margin of 16px all over the content (left, right, top, bottom)
  • Class ab-margin-top-16: creates a top margin of 16px
  • Class ab-margin-bottom-16: creates a bottom margin of 16px
  • Class ab-margin-left-16: creates a left margin of 16px
  • Class ab-margin-right-16: creates a right margin of 16px
  • Class ab-margin-24: creates a margin of 24px all over the content (left, right, top, bottom)

Syntax highlight

Angrybee dark mode includes the highlightjs library to enable the syntax highligthing for the blocks of code.

To create a block of code, simply create the following structure:

Example

Tabs control

@Todo: vertical tabs, tab header - W3.CSS How To

You can create only one "Tabs" component by HTML page in the meantime.

We create tabs using mainly div and button tags:

  • The first div includes a set of buttons (headers of tabs)
  • The following div tags represent the content of the tabs

Example to create tabs header

Here we create 2 tab headers, one with a visible title Dark mode, the other with title Paragraph.

The root div must have a class with value tab and each child button a class with value tablinks.

Each time the user click on the title, each time it show a specific content. Interaction is described in onclick attribute calling openTab() method. The 2nd parameter of the method is the value of the target div id attribute

					
<div class="tab">
  <button class="tablinks" onclick="openTab(event, 'exDark')" id="exDefaultOpen">Dark mode</button>
  <button class="tablinks" onclick="openTab(event, 'exParagraph')">Paragraph</button>
</div>

To activate a tab content by default, add id attribute with a specific value, here exDefaultOpen, to the button/content you want to display by default.

Add also, at the end of the HTML page, the following script to execute the method of the onclick attribute corresponding to the id with value exDefaultOpen.

					
<script>
	//Get the element with id="exDefaultOpen" and click on it
	document.getElementById("exDefaultOpen").click();
</script>

Example to create tabs content

For each button, you create a div with class value tabcontent to represent the content with the id value corresponding to the second parameter of the openTab() method.

					
<div id="exDark" class="tabcontent">
	<!-- insert tab content -->
</div>

<div id="exParagraph" class="tabcontent">
	<!-- insert tab content -->
</div>

Block containers

Containers are the basic UI elements to provide common margin (16px), padding (16px) and alignments.

A container is declared with ab-container.

Containers are basically displayed in row layout (like display:block style) taking 100% width of the viewport.

Example of 3 block containers



<div class="ab-container ab-green">
<p>Lorem ipsum dolor sit amet.</p>
</div>

<div class="ab-container ab-yellow">
<p>Lorem ipsum dolor sit amet.</p>
</div>

<div class="ab-container ab-orange">
<p>Lorem ipsum dolor sit amet.</p>
</div>

Inline containers

The ab-inline-grid class added to a container allows us to be displayed inline (in the same row).

ab-inline-grid has the same characteristics than display:inline-block: it allows you to align containers in the same row but with respect of width, height, padding and margin.

Example of inline containers with sufficient space

							

<div style="width: 400px;" class="ab-dark-light ab-padding-24">
	<div class="ab-container ab-inline-grid ab-width-100 ab-height-200 ab-green"></div>
	<div class="ab-container ab-inline-grid ab-width-100 ab-height-200 ab-yellow"></div>
	<div class="ab-container ab-inline-grid ab-width-100 ab-height-200 ab-orange"></div>
</div>

In this case, the viewport has the sufficient width to display all the containers inline.

Results


Example of inline containers without sufficient space

							

<div style="width: 400px;" class="ab-dark-light ab-padding-24">
	<div class="ab-container ab-inline-grid ab-width-100 ab-height-200 ab-green"></div>
	<div class="ab-container ab-inline-grid ab-width-100 ab-height-200 ab-yellow"></div>
	<div class="ab-container ab-inline-grid ab-width-100 ab-height-200 ab-orange"></div>

In this case, the viewport has not the sufficient width to display all the containers inline. The last container take the next row.
