Kick start

CSS declaration

The Angrybee Web framework is mainly based upon classes starting always by ab-....

The Angrybee Web framework is responsive and dark mode.

We propose 1 CSS adapted to all screen sizes with adaptative CSS classes for:

  • screen < 576px
  • 577px < screen < 768px
  • 769px < screen < 992px
  • 993px < screen < 1200px
  • 1201px < screen < 1400px
  • 1401px > screen

You don't need to deal with screen sizes, classes are applied depending on the screen sizes.


Angrybee CSS declaration in HTML


<link rel="stylesheet" href="$url/angrybee-dark-pc.css">

Javascript declaration

We propose also a set of Javascript libraries, designed by Angrybee but also by other Open Source projects to enhance the power of our framework and avoid reinventing the wheel.

Block containers

Containers are the basic UI elements to provide common padding (5px) and alignments. It does not provide margin to allow you to adapt easily the margin depending on the UI context.

A container is declared with ab-container class.

By default containers are basically displayed in row layout (like display:block style) taking 100% width of the viewport, but you can enforce its dimension with predefined classes.

See Predefined size section for additional information.

Example: 3 block containers


                        
<div class="ab-container ab-green">
<p>Lorem ipsum dolor sit amet.</p>
</div>

<div class="ab-container ab-yellow">
<p>Lorem ipsum dolor sit amet.</p>
</div>

<div class="ab-container ab-orange">
<p>Lorem ipsum dolor sit amet.</p>
</div>

Result

Lorem ipsum dolor sit amet.

Inline containers

If you add ab-inline-grid class to ab-container class, you can display them inline (i.e. in a single row).

ab-inline-grid is the equivalent of display:inline-block: it allows you to align containers in the same row but with respect of width, height, padding and margin. Means that you need sufficient space for all your containers to display them in one row.

If not, the containers which is over the viewport width jump to the next row.

Example: inline containers with sufficient space


                        
<div style="width: 400px;" class="ab-dark-light ab-padding-24">
    <div class="ab-container ab-inline-grid ab-width-100 ab-height-200 ab-green"></div>
    <div class="ab-container ab-inline-grid ab-width-100 ab-height-200 ab-yellow"></div>
    <div class="ab-container ab-inline-grid ab-width-100 ab-height-200 ab-orange"></div>
</div>

In this case, the viewport has the sufficient width to display all the containers inline.

Result


In the second case, the viewport has not the sufficient width to display all the containers inline. The last container take the next row.


Example: inline containers without sufficient space


                        
<div style="width: 400px;" class="ab-dark-light ab-padding-24">
    <div class="ab-container ab-margin-5 ab-inline-grid ab-width-100 ab-height-200 ab-green"></div>
    <div class="ab-container ab-margin-5 ab-inline-grid ab-width-100 ab-height-200 ab-yellow"></div>
    <div class="ab-container ab-margin-5 ab-inline-grid ab-width-100 ab-height-200 ab-orange"></div>
    <div class="ab-container ab-margin-5 ab-inline-grid ab-width-100 ab-height-200 ab-blue"></div>
</div>

By default, the row take 100% of screen or viewport width and each row item has the same size.

Grid layout

The Grid layout model offers a grid-based layout system, with rows and columns.

We propose a set of predefined:

  • rows with iso-sized columns, from 2 to 12 columns per row, using classes from ab-grid-2 to ab-grid-12
  • rows starting with fixed size columns, from 50px to 250px, growing by 50px steps, using classes from ab-grid-50-rest to ab-grid-250-rest
  • rows ending with fixed size columns, from 50px to 250px, growing by 50px steps, using classes from ab-grid-rest-50 to ab-grid-rest-250
  • rows starting and ending with fixed size columns, from 50px to 250px, growing by 50px steps, using classes from ab-grid-50-rest-50 to ab-grid-250-rest-250
